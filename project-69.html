❌ WHAT SHOULD NOT DO

1. DO NOT optimize blindly without monitoring impact
   → "Reduce all Glue jobs to 2 DPU" → jobs fail or take 3x longer
   → ALWAYS: reduce gradually (10 → 7 → 5) → monitor → adjust
   → One change at a time → measure → next change

2. DO NOT delete S3 data without confirming it's truly unused
   → "Delete everything older than 90 days"
   → Annual audit needs 12 months of data
   → Compliance may require 7-year retention
   → ALWAYS: lifecycle to Glacier first → delete only after confirmation

3. DO NOT buy Reserved Instances before understanding usage patterns
   → RI = 1-3 year commitment
   → If workload changes: locked into unused capacity
   → FIRST: 30 days of Cost Explorer data → THEN: RI decision
   → Start with: Savings Plans (more flexible than RIs)

4. DO NOT remove NAT Gateway without adding VPC endpoints
   → Glue jobs need: S3, Glue service, CloudWatch Logs, STS
   → Remove NAT without endpoints → ALL Glue jobs fail
   → Add endpoints FIRST → test → THEN remove NAT

5. DO NOT cut monitoring to save money
   → "CloudWatch costs $65 — turn it off"
   → Without monitoring: can't detect problems, can't measure savings
   → REDUCE verbosity (DEBUG → INFO) but KEEP essential metrics
   → Monitoring pays for itself in incident prevention

6. DO NOT sacrifice reliability for cost savings
   → "Remove retry logic to reduce Step Function transitions"
   → One failed pipeline = hours of manual recovery
   → Quality gates + retry logic: cost cents, save hours
   → NEVER optimize away reliability features

7. DO NOT optimize once and forget
   → Costs drift: new data sources, growing volumes, new features
   → Optimization is CONTINUOUS, not one-time
   → Monthly review cadence prevents drift
   → Set calendar reminder: "First Monday — cost review"

8. DO NOT compare AWS costs to on-premises naively
   → "On-prem server costs $500/month — AWS is $1,500"
   → On-prem doesn't include: power, cooling, staff, DR, scaling
   → True comparison: TCO (Total Cost of Ownership) analysis
   → AWS advantage: pay for what you use, scale on demand
